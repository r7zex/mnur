# МНУР

МНУР (Министерство национальной устойчивости и управления рисками) — набор моделей, сервисов и UI-компонентов для прототипа приложения мониторинга рисков, уведомлений и ресурсного планирования.

## Возможности

- Модели данных для рисков, инцидентов, укрытий и пользователей.
- Сервисы для работы с рисками, погодой, уведомлениями, геолокацией и локальной базой данных.
- UI-компоненты и тема приложения на базе [Flet](https://flet.dev/).

## Структура проекта

- `src/models` — доменные модели (риски, инциденты, укрытия, пользователи).
- `src/services` — бизнес-логика и интеграционные сервисы.
- `src/ui` — тема и UI-компоненты.
- `src/config` — настройки приложения.
- `web` — статические материалы для демо и прототипирования.

## Установка

1. Создайте виртуальное окружение Python и активируйте его.
2. Установите зависимости из `requirements.txt`.

## Быстрый старт

Для просмотра прототипа интерфейса сначала запустите локальный сервер статических файлов:

1. Запустите `src/web_server.py`. По умолчанию сервер поднимется на `http://127.0.0.1:8000`.
2. Откройте адрес в браузере — сервер отдаёт содержимое каталога `web` и конфигурацию `config.js`.

После этого можно интегрировать модули проекта в своё приложение: модели находятся в `src/models`, сервисы — в `src/services`, UI-компоненты — в `src/ui/components`.

## Конфигурация

Базовые настройки расположены в `src/config`. Для рабочей интеграции рекомендуется:

- хранить ключи и секреты в переменных окружения;
- использовать файл `.env` или менеджер секретов в зависимости от инфраструктуры;
- документировать требуемые значения в README конкретного сервиса/модуля.
